# proxy-printer skill — design

Date: 2026-05-09

## Purpose

A new skill that turns a parsed MTG deck list into printable proxies:
one PDF for the deck's cards, one PDF for every token kind those cards
produce. Both PDFs use a layout that mirrors a real MTG card frame —
name banner, ASCII art, type banner, oracle text, P/T — and pull all
card data live from Scryfall. No training-data oracle text, ever.

The skill packages the ad-hoc workflow we used in `/tmp/proxies-322815/`
into a reusable, testable surface that any of the other skills
(deck-wizard, cube-wizard) can chain against.

## Scope

- One CLI: `proxy-print --kind cards|tokens --deck DECK.json --out OUT.pdf`.
- Single combined skill (cards + tokens). One invocation per kind so
  callers can re-render only what changed.
- Input is the canonical parsed-deck JSON that `parse-deck` produces.
  The SKILL.md tells the agent to invoke `parse-deck` first.
- Output is two PDFs (one per kind). Letter (default) or A4. 3×3 grid
  of 2.5″×3.5″ cards.
- Art catalog covers card subtypes, not individual card names. Same
  catalog used for both cards and tokens.

## Non-goals (v1)

- Cut guides / crop marks
- Image-based proxies (Scryfall art download)
- Foil indicator
- Double-faced card back rendering
- Custom / homebrew tokens not in Scryfall
- Per-card art overrides

## Architecture

Approach A from the brainstorm: new skill directory at the repo root +
shared CLI in mtg-utils. Mirrors deck-wizard / cube-wizard / rules-lawyer
/ lgs-search exactly. Rejected:

- **B (CLI-only)** — no SKILL.md means no agent-facing guidance and the
  skill never appears in skill discovery.
- **C (fold into deck-wizard)** — proxies are *output*, not building or
  tuning, and the token discovery walks any card list (cubes too). Mixing
  the concern bloats deck-wizard's already 1883-line SKILL.md.

## CLI surface

```
proxy-print --kind {cards,tokens} --deck DECK.json --out OUT.pdf [options]
```

**Required**

- `--kind cards|tokens` — which PDF to render this invocation.
- `--deck PATH` — parsed deck JSON (`parse-deck` output schema).
- `--out PATH` — output PDF path.

**Optional**

- `--bulk-data PATH` — Scryfall bulk JSON. Defaults to the standard
  download dir (`download-bulk` writes there).
- `--page-size letter|a4` — default letter.
- `--copies N` — extra copies of every card / token (default 1). For
  cards this multiplies on top of deck quantities.
- `--include-sideboard / --no-sideboard` — cards mode only; default
  include.
- `--report-art-coverage` — tokens mode only; emits per-token JSON to
  stderr showing which catalog tier hit. Useful for finding gaps.

**Exit codes**

| Code | Cause                              |
|------|------------------------------------|
| 0    | success                            |
| 1    | bulk data missing / stale          |
| 2    | deck JSON missing / malformed      |
| 3    | output path unwritable             |
| 4    | rendering failure                  |

## Layout

Cards and tokens share one render template. Differences are in (a) whether
mana cost appears in the name banner, and (b) how much vertical space goes
to oracle text vs. art.

```
┌───────────────────────────────┐
│ ╔══════════════════════════╗  │  name banner
│ ║ NAME              {COST} ║  │  (cost omitted for tokens)
├─╚══════════════════════════╝──┤
│                               │
│         (ASCII art)           │  art region (proportional)
│                               │
├───────────────────────────────┤
│ ╔══════════════════════════╗  │  type banner
│ ║ Type — Subtype       {C} ║  │
├─╚══════════════════════════╝──┤
│ oracle text                   │  oracle region (proportional)
│                       ┌─────┐ │
│                       │ P/T │ │
└───────────────────────┴─────┴─┘
```

**Geometry**

- Card size: 63.5mm × 88.9mm (2.5″ × 3.5″).
- 3 columns × 3 rows = 9/page on Letter (8.5″ × 11″) or A4 (210×297mm).
- 0.5″ horizontal / 0.25″ vertical outer margins on Letter; centered on A4.
- Outer card border: 0.6pt black stroke (cut line).

**Banner styling.** Light-gray fill (0.92), thin black stroke, ~14pt
tall. Reportlab `rect(fill=1, stroke=1)` followed by overlaid text.

**Body proportion algorithm.**

1. Pre-measure oracle text at 7.5pt to estimate height needed.
2. Reserve that height (capped at 60% of body) for oracle.
3. Remaining body height → art region (clamped to ≥ 40pt).

Result: token cards (no oracle) get a huge art region; spell cards get a
small art region above a substantial oracle box; lands somewhere between.

**Fonts.** Helvetica for text, Courier-Bold for ASCII art. Both reportlab
built-ins — no font files to ship.

**Auto-fit.** Oracle text shrinks 9.5pt → 6.5pt (cards) or 7.5pt → 5.5pt
(token oracle box). ASCII art shrinks Courier-Bold 8.0pt → 5.5pt to fit
the allocated band.

**P/T box.** Drawn for creatures (`power` and `toughness` both present);
loyalty rendered as `L: <n>` for planeswalkers; absent for everything
else. ~0.55″ × 14pt at the bottom-right.

**Tokens-only `from:` footer.** Italic 6pt to the left of the P/T box,
truncated with ellipsis. `from: <name>` for one source, `from: N cards`
for multiple.

**Mana cost rendering.** Scryfall braces, e.g. `{2}{B}{R}`. Symbol-font
rendering is out of scope — proxy is unambiguous either way.

## ASCII art catalog

**One `.txt` per card subtype, slug-cased**, plus card-type fallbacks
and an ultimate generic. Catalog lives at
`mtg-utils/src/mtg_utils/data/card_art/`.

**Composition (~210 files)**

- ~150 creature subtypes (vampire, knight, soldier, beast, dragon, …)
- ~20 artifact subtypes (equipment, vehicle, treasure, food, clue, …)
- ~15 enchantment subtypes (aura, curse, saga, shrine, class, …)
- 5 basic lands + ~10 land subtypes (plains, island, …, desert, gate, …)
- 7 card-type fallbacks (creature, artifact, enchantment, land, sorcery,
  instant, planeswalker)
- 1 ultimate fallback (`_generic.txt`)

Art is **P/T-independent** — all 1/1 / 2/2 / 3/3 Soldier cards share
`soldier.txt`. P/T is rendered as text.

**Lookup algorithm** (same for cards and tokens):

1. Parse `type_line`. Split on `—`. Card-types on the left
   ("Legendary Creature"), subtypes on the right ("Vampire Knight").
2. For each subtype word in order, try `<slug>.txt`. First hit wins.
3. If no subtype matched (or no subtypes existed), iterate card-type
   words and try `<card-type-slug>.txt`. Skip non-art words: "Token",
   "Legendary", "Snow", "Tribal", "Basic".
4. Final fallback: `_generic.txt`.

**Examples**

| Card                                       | Lookup chain                         | Hits        |
|--------------------------------------------|--------------------------------------|-------------|
| Toxic Deluge (Sorcery)                     | sorcery                              | sorcery     |
| Skullclamp (Artifact — Equipment)          | equipment → artifact                 | equipment   |
| Tyvar (Legendary Creature — Elf Warrior)   | elf → warrior → creature             | elf         |
| Forest (Basic Land — Forest)               | forest → land                        | forest      |
| Sulfurous Springs (Land)                   | land                                 | land        |
| Vampire Knight token                       | vampire → knight → creature          | vampire     |
| Treasure token (Token Artifact — Treasure) | treasure → artifact                  | treasure    |

**Slug normalization.** Lowercase. Strip apostrophes. Replace
non-alphanumerics with `-`. Collapse runs of `-`. Examples:
`Eldrazi Spawn → eldrazi-spawn`, `Urza's → urzas`, `Phyrexian Mite → phyrexian-mite`.

**Loading.** `importlib.resources.files("mtg_utils.data.card_art")` so
the catalog works equally from a checkout, an editable install, and a
wheel.

**Coverage report.** `--report-art-coverage` writes one JSON line per
token to stderr: `{"name": ..., "tier": "subtype|card-type|generic", "key": "knight"}`.
The user can then prioritize gaps.

## Token discovery

Per deck card:

1. Look up the source card by lowercased name in a bulk index. Skip
   with a `WARN: missing from bulk:` if absent.
2. Read `all_parts` from the source. Filter to entries with
   `component == "token"`.
3. For each token-part, look up the full token by `id` in a separate
   `by_id` index built from the same bulk pass that keeps `layout: token`
   entries (the standard `_load_bulk_index` filters them out).

A separate `by_id` index is necessary because
`scryfall_lookup._extract_fields` strips fields the renderer needs
(`power`, `toughness`, `loyalty`, `card_faces`, `all_parts`).

**Dedup key.** `oracle_id` (Scryfall's stable cross-printing identifier).
Falls back to `(name, type_line, power, toughness)` if `oracle_id` is
missing. Two printings of the same token kind collapse correctly.

**Source attribution.** Each token group accumulates a list of source-card
names. Footer renders `from: <name>` for one, `from: N cards` for many.

**Sort order on the page.** Artifacts first, then color groups
(W → U → B → R → G → C), then alphabetical by name. Stable across runs.

**Edge cases.**

- Source has `card_faces` and no top-level `oracle_text` (split / MDFC) —
  walk `all_parts` from each face. Most MDFCs put `all_parts` at the top
  level so this is a small case.
- Custom / homebrew tokens — out of scope. Documented limitation.

## Card walk

Cards mode iterates `commanders` + `cards` + (`sideboard` unless
`--no-sideboard`). Quantities expanded to one slot per copy. `--copies N`
multiplies. Same Scryfall hydration as token discovery (lookup by
lowercased name, fall back to a `WARN:` and skip if missing from bulk).

## Errors and edge cases

**Bulk data resolution**

1. `--bulk-data` flag if given.
2. Else: `download-bulk`'s standard output dir
   (`$MTG_SKILLS_CACHE_DIR/scryfall-bulk/` or `/tmp/scryfall-bulk/`).
3. If missing or > 7 days old → exit code 1, stderr message points the
   user at `download-bulk`. Don't silently fetch ~500MB from a render
   command.

**Hard errors → exit codes 1–4** as per the CLI table.

**Soft warnings → stderr, exit 0**

- Source card missing from bulk → `WARN: missing from bulk: <name>`. Skip.
- Token id in `all_parts` not in bulk → `WARN: token id <id> from <src>`. Skip.
- Deck-list quantity > 99 → safety warning; likely malformed JSON.

**Logging.** Stderr only, prefixed lines (`OK:`, `WARN:`, `MISS:`).
PDF path is the only stdout line on success — shells can capture it.

**Concurrency.** Not pursued. Each invocation builds its own in-process
bulk index from the same atomic-rename pickled sidecar; no shared
mutable state. `bulk_loader`'s existing write path already handles
concurrent readers.

**Determinism.** Outputs are deterministic given (bulk file, deck JSON,
flags). Sort keys are total. Catalog files read in stable iteration order.

## Package data

```toml
# mtg-utils/pyproject.toml
[tool.setuptools.package-data]
"mtg_utils.data.gauntlets" = ["*.json"]
"mtg_utils.data.card_art"  = ["*.txt"]
```

Both `data/__init__.py` and `data/card_art/__init__.py` are required so
setuptools treats them as packages.

`proxy-printer/pyproject.toml` re-declares the `proxy-print` entry point
(it does not re-declare `parse-deck` etc. — the SKILL.md tells the agent
to invoke `parse-deck` separately, so users running just the proxy CLI
don't need every deck-wizard tool installed).

## Testing

Layout: `tests/proxy-printer/test_*.py`.

**Categories**

1. **Slug + lookup** — `slug()` correctness, lookup chain tier ordering,
   skipped non-art words.
2. **Token discovery** — in-memory bulk fixture, expected
   `(oracle_id, sources)` groups, dedup correctness, sort stability.
3. **Card walk** — quantities expand, sideboard flag respected,
   commanders always included.
4. **Render smoke** — render a 5/12/20-card deck, open with pypdf,
   assert page count + names appear in extracted text. Same for tokens.
5. **Catalog coverage** — walk full bulk, list every distinct subtype,
   assert each has either a `.txt` file or a documented fallback.
   Skipped in CI (depends on local bulk); emits a TestReport gap list.
6. **CLI** — `--help` exit 0, missing required flags exit non-zero,
   stale bulk → exit 1 + stderr message mentions `download-bulk`.

**Mocking.** `unittest.mock.patch` on `_api_lookup` so any path that
would hit Scryfall fails loudly. Tests pass an in-memory `bulk_index`
to the existing `lookup_single` API.

**No real network in tests.** Fixtures are tiny dict literals.

**Golden PDFs.** Deferred — font rendering varies across reportlab
versions and the file format is brittle to byte-equal compare. Layout
sanity is via extracted-text + page-count assertions instead.

## SKILL.md outline (to be written by writing-plans)

The SKILL.md should teach the agent:

- Iron rule: every proxy renders oracle text from Scryfall, never from
  training data. (Same iron rule as deck-wizard / cube-wizard.)
- The chain: parse-deck → proxy-print --kind cards → proxy-print --kind tokens.
- When invoked by another skill (deck-wizard's "print this list" path),
  expect a parsed deck JSON path; never invent the schema.
- How to interpret `WARN:` output (don't escalate; tell the user the
  card is missing and continue).
- Cross-skill use: cube-wizard can also call proxy-print on a parsed
  cube-as-deck JSON — the schema is compatible.
